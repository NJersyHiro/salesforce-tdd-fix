# FCチームへようこそ

## セットアップチェックリスト

### 1. Claude Code のインストール

```bash
npm install -g @anthropic-ai/claude-code
```

### 2. MCPサーバーの設定

ターミナルで以下を実行してください：

```bash
# Backlog — チケット取得・ステータス更新
claude mcp add backlog

# Notion — 修正記録ページの作成・更新
claude mcp add notion

# Playwright — vpartial環境でのバグ再現・ST実行
claude mcp add playwright
```

- [ ] **Backlog** — チームのBacklogスペースへのアクセス権限が必要です。管理者に招待を依頼してください。
- [ ] **Notion** — チームのNotionワークスペースへのアクセス権限が必要です。管理者に招待を依頼してください。
- [ ] **Playwright MCP** — ローカルインストール後すぐ使えます。

### 3. スキルのインストール

```bash
# スキルディレクトリに移動
cd ~/.claude/skills

# salesforce-tdd-fix スキルを配置
# （チームの共有リポジトリまたはNJersyHiroから受け取ってください）
```

- [ ] `salesforce-tdd-fix` — Salesforceバグ修正の主要ワークフロー（12ステップ）
- [ ] `salesforce-st-execution` — STシナリオ実行専門スキル

### 4. Salesforce CLI

```bash
# Salesforce CLI のインストール（未インストールの場合）
npm install -g @salesforce/cli

# vpartial 環境への認証
sf org login web --alias vpartial
```

- [ ] Salesforce CLI インストール済み
- [ ] vpartial 環境へのログイン済み

## salesforce-tdd-fix スキルの使い方

Backlogでバグチケット（例：`COMPASS-1234`）を受け取ったら、Claude Codeを開いて以下を入力するだけです：

```
COMPASSのチケットXXXXのバグを修正したい
```

Claudeが自動的に `salesforce-tdd-fix` スキルを呼び出し、以下の12ステップをガイドします：

| # | ステップ | あなたがすること |
|---|---------|----------------|
| 1 | 要件把握 | チケット番号を伝えるだけ |
| 2 | バグ再現 | Claudeがvpartialでブラウザ操作 |
| 3 | テストシナリオ作成 | Claudeが提案、あなたが確認 |
| 4 | テストデータ確認 | ClaudeがSOQLでデータ検索 |
| 5 | 修正方針の確認 | Claudeが方針を提示 |
| **6** | **あなたの承認** ← **ここで必ず確認** | 「問題なければ実装を開始します」に返答する |
| 7 | Notion文書化 | 自動で記録ページ作成 |
| 8 | 実装＋デプロイ | Claudeが実装しvpartialにデプロイ |
| 9 | バックアップ | 自動でバックアップ取得 |
| 10 | STシナリオ実行 | Claudeがブラウザで全シナリオ実行 |
| 11 | スクショ保管 | 自動で整理・保管 |
| 12 | クリーンアップ＋PR作成 | 自動でチケットクローズ・PR作成 |

> **重要：** Step 6はHARD STOPです。Claudeは必ずここで止まり、あなたの承認を待ちます。実装は承認後にしか開始されません。

## チームTips

- **STは必ず営業担当ユーザーで実施する** — システム管理者は権限が広すぎて本番と条件が異なります。Claudeが切り替え手順をガイドします。
- **`git add .` は使わない** — リトリーブで混入したメタデータ差分を誤って含めないよう、Claudeは常にファイルを個別に指定します。
- **作業ブランチは `Sprint-FY26-2_YAMAMOTO` 固定** — 新規ブランチは作らず、このブランチを使い続けます。
